# SnapParse 使用说明（中文）

本文档面向最终用户与贡献者，介绍 SnapParse 的主要功能、配置方式与使用建议。

## 1. 软件定位

SnapParse 是一款 Windows 桌面效率工具，聚合了：

- 剪贴板历史管理
- 划词助手（复制、总结、优化、解释、翻译、搜索、自定义 Agent）
- 智能 OCR（框选屏幕识别文本）
- TTS 文本朗读

## 2. 核心界面

### 2.1 剪贴板窗口

- 默认为竖向列表，按时间展示采集内容
- 支持类型筛选：全部、图片、链接、文本、收藏
- 点击条目可复制到系统剪贴板（可配置是否自动关闭窗口）
- 支持收藏切换（用于后续快速筛选）

### 2.2 划词条形栏

- 在检测到文本选区后，在选区附近弹出功能条
- 可执行动作：
  - 复制
  - 总结
  - 优化
  - 解释
  - 翻译
  - 搜索
  - 自定义 Agent
- 支持紧凑模式（仅图标）
- 支持显示项排序、开关、数量限制

### 2.3 处理窗口（划词 / OCR）

- 展示模型处理结果，支持流式输出
- 常见操作：复制、朗读、收藏、置顶、关闭
- 原文区支持折叠/展开
- 翻译窗口支持源语言（自动检测）和目标语言选择

### 2.4 OCR 处理窗口

- 上半部分显示 OCR 原文
- 下半部分显示处理结果（默认翻译，可在设置切换为总结/优化/自定义）
- 支持分别复制原文与结果

## 3. 设置项说明

### 3.1 通用设置

- 主题
- 语言
- 开机自启
- 静默启动
- 窗口失焦关闭（全局）
- 记忆窗口大小（全局）
- 记忆窗口位置（全局）

### 3.2 剪贴板设置

- 采集条目数量
- 轮询间隔
- 点击条目复制后自动关闭
- 顶栏折叠行为
- 存储位置（支持选择文件夹）

### 3.3 划词助手

- 启用/禁用各功能项
- 条形栏功能排序
- 紧凑模式
- 显示时长
- 忽略应用名单（按可执行文件名匹配）
- 自定义 Agent（名称、图标、Prompt）

### 3.4 智能 OCR

- OCR 模型 API 配置
- OCR 后默认处理方式
- 记忆 OCR 处理窗口大小

### 3.5 TTS 设置

- 中文语音选择（下拉）
- 英文语音选择（下拉）
- 语速、音量（如已提供）

### 3.6 快捷键 / 关于与诊断 / 数据备份

- 快捷键：管理唤起剪贴板、OCR 等全局按键
- 关于与诊断：版本、运行状态、检查更新与安装更新
- 数据备份：导入导出配置与数据

## 4. 模型配置建议

### 4.1 LLM（总结/优化/解释/翻译）

- 使用 OpenAI 兼容接口时，确保：
  - Base URL 正确
  - API Key 有效
  - 模型名可用
  - 超时时间与 token 上限合理

### 4.2 OCR（以 GLM OCR 为例）

- 请求需使用模型要求的图片格式（JPG/PNG/PDF）
- 返回内容可能在字段 `md_results` / `layout_details` 中
- 若识别文本前缀有 `##`，客户端会做清洗显示

## 5. 常见问题

### 5.1 为什么有时划词后不弹条形栏？

- 先检查“忽略应用名单”是否误匹配当前应用
- 检查条形栏显示时长是否过短
- 检查该功能是否被关闭或排序后超出显示上限

### 5.2 OCR 识别成功后又变失败怎么办？

- 检查是否触发重复请求覆盖状态
- 检查接口超时与并发控制
- 查看设置中的 API 响应日志

### 5.3 开机自启状态显示不一致

- 需同时校验系统注册状态与软件内部配置
- 建议重启后再次进入设置页面确认状态同步

- 发布前统一版本号（`package.json`、`tauri.conf.json`、`Cargo.toml`）
- 打包 `msi` 与 `nsis` 两种安装形式
- 在 GitHub Releases 附上：
  - 安装包
  - 更新说明
  - 已知问题

更新机制说明见：`docs/RELEASE_AND_UPDATE.md`
